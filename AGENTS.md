# AGENTS.md

The rules any coding agents working on this project should follow.

- Before implement a code snippet, function, or method, search the similar approach in the codebase first. Do not add dupliated codes.
- Before you conclude you complete the task requested, run lint, formatter, and tests.
- Do not use future statement definitions such as `from __future__ import annotations` because we are using Python 3.14+ versions.
- Write the code with proper typing, and simple and concise naming.
- Do not generate unnecessary comments and docstrings. Most of cases are enough with proper typing and naming. Write the comments if the related code is irregular due to some reasons or constraints.
